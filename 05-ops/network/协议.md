应用层协议

## 响应状态码
* 信息相应（100-199）
    * 100 continue
    * 101 switch protocols
    * 102 processing（webdav）
    * 103 Early Hints
        服务相应之前，想服务端发送服务端可能用到的提示信息。以允许代理在服务准备相应节点开始预加载。
* 成功相应（200-299）
    * 200 OK
    * 201 Created
        请求成功，并因此创建了一个新的资源。（POST，PUT）
    * 202 Accepted
        请求已收到，还未相应，没有结果。
    * 203 Non-Authoritative Information
        服务器已经成功处理请求，但返回的实体头部元信息不是来自原始服务器，而是来自本地或者第三方的副本。也就是说服务器从一个不同的源获取了相应数据，并返回给客户端。
    * 204 No content
        对于该请求没有内容可发送，但头部字段可能有用。
    * 205 Reset Content
        告诉客户代理重新发送此请求文档。
    * 206 Partial Content
        客户端发送range范围的表头以请求资源的一部分时。
    * 207 Multi-Status（WebDAV（Web Distributed Authoring and Versioning））
        对于多个状态码都可能合适的情况，传输有关的多个资源信息。
    * 208 Already Reported（WebDAV）
        DAV里面使用<dav:propstat>相应元素以避免重复枚举多个绑定的内部成员到同一个集合。
    * 226 IM Used（http delta encoding）
        服务器已完成了对资源的GET请求，并且响应是对当前实例应用的一个或多个实例操作的表示。
* 重定向消息（300-399）
    * 300 Mutiple Choics
    * 301 Moved Permanently
        永久移动，给出新位置
    * 302 Found
    * 304 Not Modifed
    * 305 Use Proxy
    * 306 unused 
        已弃用
    * 307 Temporary Redirect
    * 308 Permanent Redirect
* 客户端错误（400-499）
    * 400 Bad Request
    * 401 Unauthorized （重点，开源中国面过）
    * 402 Payment Required
    * 403 Forbidden （重点,开源中国面过）
        * 与401异同
            * 相同点
                都是没有权限
            * 不同点
                403 知道用户的身份
    * 404 not found
    * 405 Method Not Allowed
    * 406 Not Acceptable
    * 407 Proxy Authentication Required
        与401类似，但是认证需要由代理完成。
    * 408 Request Timeout
    * 409 conflict
    * 410 Gone
        请求内容已从服务器中永久删除，并没有转发地址。
    * 411 Lenth Required 
        拒绝该请求，Content-Length头部字段未定义但是服务端需要他。
    * 412 Precondition Failed
        客户端在请求头文件中指出了服务器不满足先决条件。
    * 413 Payload Too Large
        请求实体大于服务器定义的限制。
    * 414 URI too long
    * 415 Unsupported Media Type
    * 416 RangeNot Satisfiable
    * 417 Expectation Failed
    * 418 I'm a teapot
* 服务端错误（500-599）
    * 500 Internal Server Error
    * 501 NotImplemented
    * 502 Bad Gateway
    * 503 service Unavailable
    * 504 Gateway Timeout
    * 505 Http Version Not Supported
    
# https
http的安全版本，通过http添加ssl、tls协议来实现数据加密传输和安全传输。确保数据在客户端和服务端之间传输的机密性、完整性和身份验证。

# WebSocket
全双工双向通信协议，基于 TCP，在单一 TCP 连接中提供全双工通信能力。允许服务器主动向客户端推送数据。

## 特点
* **实时双向通信**：支持服务器和客户端之间的实时双向数据交互，无需轮询
* **低延迟**：建立连接后，数据通过已有连接传输，避免频繁建立连接的开销
* **单一连接**：一个连接可以承载多条独立的逻辑流
* **减少开销**：相比 HTTP 轮询，减少了大量的头部信息和连接重建的开销
* **支持 HTTP 升级**：通过 HTTP Upgrade 机制从 HTTP 升级到 WebSocket

## 与 HTTP 的区别
| 特性 | HTTP | WebSocket |
|-----|------|-----------|
| 通信方式 | 单向（请求-响应） | 双向（全双工） |
| 连接持久性 | 短连接或 Keep-Alive | 长连接 |
| 数据推送 | 需要轮询 | 服务器主动推送 |
| 延迟 | 较高 | 低延迟 |
| 开销 | 每次请求都有 HTTP 头 | 建立连接后开销小 |

## 工作流程
1. **握手阶段**：
   - 客户端发送 HTTP GET 请求，包含 `Upgrade: websocket`、`Sec-WebSocket-Key` 等头部
   - 服务器确认升级，返回 HTTP 101 状态码（Switching Protocols）
   - 升级成功后，连接变为 WebSocket 协议

2. **数据传输阶段**：
   - 建立 TCP 连接，收发基于 WebSocket 帧（Frame）的数据
   - 支持文本帧（text frame）和二进制帧（binary frame）

3. **关闭阶段**：
   - 任一方可发送关闭帧（close frame）
   - 接收方确认后关闭连接

## 应用场景
* **实时通信**：即时聊天、多人协作编辑
* **实时数据推送**：股票行情、消息推送、实时通知
* **游戏**：多人在线游戏的实时状态同步
* **IoT**：设备实时数据上报和远程控制
* **监控告警**：实时监控面板、系统日志流推送
* **直播互动**：弹幕、礼物、评论实时同步

## 安全性
* 使用 WSS（WebSocket Secure，基于 TLS/SSL）实现加密传输
* 通过 `Sec-WebSocket-Key` 验证和 `Origin` 头防止跨站攻击
* 需要在应用层实现额外的身份验证和授权

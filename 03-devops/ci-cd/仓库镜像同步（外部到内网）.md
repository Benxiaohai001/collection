# 仓库镜像同步（外部到内网）

## 适用场景

将外部 Git 仓库完整镜像（包含全部分支、标签、引用）同步到公司内网 Git 服务。

## 脱敏版操作步骤

```bash
# 1) 克隆外部仓库镜像
git clone --mirror https://github.com/<org-or-user>/<repo>.git

# 2) 将镜像目录传到内网机器（可选，如果需要中转）
rsync -avP <repo>.git <internal-user>@<internal-host>:/path/to/workdir

# 3) 进入镜像仓库目录
cd <repo>.git/

# 4) 设置推送地址为内网仓库（仅 push URL，不影响 fetch）
git remote set-url --push origin ssh://git@<internal-git-host>:<port>/<group>/<repo>.git

# 5) 镜像推送到内网
git push --mirror
```

## 关键说明

- `--mirror` 会同步所有 refs（包含分支、tags、远端跟踪引用等），比普通推送更完整。
- `git remote set-url --push origin ...` 只改推送地址，避免误从内网地址拉取。
- 目标仓库建议提前创建空仓库，避免非预期历史冲突。

## 常用校验

```bash
# 查看推送地址是否已切换为内网
git remote -v

# 查看本地镜像 refs 数量（可与目标仓库对比）
git show-ref | wc -l

# 仅查看将推送的 refs（不真正推送）
git push --mirror --dry-run
```

## 风险与注意事项

- `git push --mirror` 具有覆盖性：目标仓库中不存在于源仓库的 refs 可能被删除。
- 首次同步建议先使用 `--dry-run`，确认无误后再正式推送。
- 涉及生产仓库时，建议在变更窗口执行，并保留操作日志。
- 若仓库很大，建议在稳定网络环境下执行，必要时分时段重试。
